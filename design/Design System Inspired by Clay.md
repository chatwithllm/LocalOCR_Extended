## 1. Visual Theme & Atmosphere

Clay is the optimistic cousin of Apple's minimalism: the same restraint with the color turned up and the corners turned round. Where Apple whispers on black-and-gray, Clay speaks on a warm cream canvas (`#faf9f7`) the color of fresh mochi, with chunky rounded rectangles that feel pressed out of modeling clay rather than machined from aluminum. Every surface has weight — you can almost feel the squish.

The typography is Roobert, a neo-grotesque designed by Displaay for Matter Supply, extended here with five OpenType stylistic sets that swap in playful alternates: single-story `a` (`ss01`), round `g` (`ss03`), open `l` / `t` terminals (`ss10`, `ss11`), and soft-shouldered `y` (`ss12`). Turn them all on at once — this typeface was built to be dressed up.

The color story is a **named swatch palette**: Matcha (green), Slushie (blue), Lemon (yellow), Ube (purple), Pomegranate (red), Blueberry (deeper blue), Dragonfruit (pink). Each swatch appears as a full-bleed section background or a large component fill; we never use them as 1-pixel accents. The accent color is Matcha (`#8fbf9f`) — it's on every primary CTA, every success state, every active nav pill.

The signature interaction is the **tilted hover**: buttons and cards rotate `-8deg`, lift `-80%` on the Y axis, and cast a hard offset shadow of `-7px 7px 0 0 #1a1a1a`. There is no blur. The shadow is a flat black rectangle behind the element, like a risograph print. This is the single most recognizable Clay motion — if you skip it, the theme is gone.

**Key Characteristics:**
- Warm cream canvas (`#faf9f7`) — never pure white, never gray
- Roobert with `font-feature-settings: "ss01", "ss03", "ss10", "ss11", "ss12"` always on
- Seven named swatches (Matcha / Slushie / Lemon / Ube / Pomegranate / Blueberry / Dragonfruit) used as surfaces, not accents
- Matcha (`#8fbf9f`) is THE interactive accent — primary CTAs, focus rings, active states
- Generous radii: 24px default, 40px for hero cards, 999px pills
- Flat hard-offset shadows (`-7px 7px 0 0 #1a1a1a`) — no blur
- Playful hover: `transform: rotate(-8deg) translateY(-80%)` + hard shadow reveal
- Multi-layer shadow with **inset highlight** (`inset 0 1px 0 rgba(255,255,255,0.6)`) for pressable feel
- Ink body text (`#1a1a1a`) on cream — warmer and less harsh than `#000`

## 3. Typography Rules

### Font Family
- **Primary**: `Roobert`, with fallbacks: `-apple-system, BlinkMacSystemFont, "Inter var", "Helvetica Neue", Helvetica, Arial, sans-serif`
- **Mono**: keep existing `--font-mono` stack (SF Mono / Menlo / JetBrains Mono)
- Load Roobert via `@font-face` in `design/colors_and_type.css` (replace the Inter `@font-face` block). If a licensed file is not present in `src/frontend/fonts/roobert/`, fall back to Inter — the stylistic-set calls are no-ops on Inter, which is fine.

### Always-on features
```css
font-feature-settings: "kern", "liga", "ss01", "ss03", "ss10", "ss11", "ss12";
```
Set this once on `:root` and inherit. Do **not** scope it to headlines — Clay's stylistic sets are just as visible at 12px.

Keep the existing 13-step scale in `colors_and_type.css` but re-tune: Clay headlines are slightly *less* tight than Apple (1.12 vs 1.07) and tracking is neutral-to-positive, not negative.

### Principles
- **Tracking is neutral.** Apple ran negative tracking at every size; Clay does not. Update every `--font-*-track` token to `0` except `hero` and `4xl` (keep slightly negative).
- **Never italicize.** Roobert has an italic cut but Clay doesn't use it — emphasis is via weight or color.
- **Uppercase sparingly.** Only `t-nano` becomes uppercase with `letter-spacing: 0.04em`.
- **No all-caps section headers.** Apple forbids them; Clay does too.

## 4. Component Stylings (mapped to actual selectors)

### Buttons

All button classes in `src/frontend/index.html` (`.btn`, `.btn-primary`, `.btn-secondary`, `.btn-ghost`, `.btn-danger`, `.btn-sm`) already consume `--color-brand` / `--color-text-primary` / `--radius-sm`. Theme via tokens + add the Clay hover.

**Primary (`.btn-primary`)**
- Background: `var(--color-brand)` = Matcha
- Text: `#1a1a1a` (ink on matcha — not white; matcha is too light for white text to meet AA)
- Padding: `12px 24px` (bump from Apple's `8px 15px` — Clay is chunkier)

- Focus: `outline: 3px solid var(--color-focus); outline-offset: 3px`

**Danger (`.btn-danger`)**

## 6. Depth & Elevation

Clay's shadow language is **hard-offset** — crisp black rectangles with zero blur. The only blurred shadow is dark-mode's cream shadow (full black would disappear).

| Level | Treatment | Use |
|-------|-----------|-----|
| Flat (0) | `none` | Body text, inline elements |
| Lift 1 (pressable) | `-3px 3px 0 0 #1a1a1a` | Small buttons, chips |
| Lift 2 (button) | `-4px 4px 0 0 #1a1a1a, inset 0 1px 0 rgba(255,255,255,0.35)` | Primary buttons rest |
| Lift 3 (card) | `-6px 6px 0 0 #1a1a1a, inset 0 1px 0 rgba(255,255,255,0.5)` | Cards |
| Lift 4 (hover reveal) | `-8px 8px 0 0 #1a1a1a` | Card hover, button hover peak |
| Lift 5 (modal) | `-12px 12px 0 0 #1a1a1a, inset 0 2px 0 rgba(255,255,255,0.5)` | Modals, dialogs |

Override the existing `--shadow-*` tokens accordingly. Consumers (`--shadow-1`, `--shadow-2`, etc.) don't need to know the language changed.

The **inset highlight** (`inset 0 1px 0 rgba(255,255,255,0.X)`) is mandatory on filled buttons and cards — it simulates the lip of a pressed-out clay tile and separates Clay from generic "brutalist" themes.

```css
    transform: none !important;
  }
}
```

## 8. Do's and Don'ts

### Do
- Always turn on all five stylistic sets (`ss01`, `ss03`, `ss10`, `ss11`, `ss12`) globally
- Use ink (`#1a1a1a`) on cream — never pure black on pure white
- Give every filled surface a 2px ink border + hard offset shadow
- Use Matcha as THE accent — primary CTAs, active nav, focus rings, success
- Use named swatches as surfaces (full button backgrounds, card left borders), not as tint layers
- Respect `prefers-reduced-motion` — disable transforms, keep color changes
- Keep the `-8deg` button tilt; it is the theme's signature
- Use the inset highlight (`inset 0 1px 0 rgba(255,255,255,0.X)`) on every filled component

### Don't
- Don't use `#000` or `#fff` anywhere — ink is `#1a1a1a`, cream is `#faf9f7`
- Don't soften the shadow with blur — hard offsets only (except the modal backdrop scrim)
- Don't use Pomegranate as an accent; it's destructive-only
- Don't repaint `.btn-ghost` (including "Not a dup") in a swatch color — user feedback confirms it stays neutral
- Don't use negative letter-spacing at small sizes (that's Apple's move)
- Don't rotate cards on hover — only buttons and the dismissed-dedup-card exit animation rotate
- Don't delete the Apple tokens — add Clay as a sibling theme so we can A/B via `[data-theme="clay"]`
- Don't introduce a gradient, texture, or glass-blur anywhere. Solid fills only.
