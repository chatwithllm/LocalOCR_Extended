How it works

Three steps. Everything on your side of the wire.

Step 01 · Drop

In from anywhere.

Phone, scanner, email forward, watched folder. Receipts land on your server and stay there.

Step 02 · Process

Locally orchestrated.

LocalOCR calls the vision API you chose, using your keys. The image goes out, JSON comes back, nothing else is logged.

Step 03 · Own

Yours to keep.

SQLite database. Text files. Original images retained. Export, back up, migrate, delete — your rules.

The privacy story

Two ways to handle a receipt.

One sends it to a SaaS. One keeps it on your box. Guess which is which.

Typical receipt app

“Cloud-based” expense trackers

  • Your receipts uploaded to their cloud, indefinitely
  • Trained on your data unless you explicitly opt out
  • Shared with “business partners” you’ve never heard of
  • Subscription goes up every April
  • Acquired by a larger company with a vaguer privacy policy
  • Goes offline when their servers do
VS
LocalOCR

On your server

  • Receipts never leave your machine except to OCR
  • Zero analytics, zero telemetry, zero trackers
  • You bring the vision API keys — or skip them
  • MIT licensed, no subscription, no rug-pull
  • Fork it if we do something dumb
  • Works offline for everything except the OCR call

Install

Running in five minutes.

One docker-compose file. No account creation, no OAuth dance, no email verification. Clone, configure your keys, bring it up.

View on GitHub Read the docs › 
# clone
$ git clone github.com/localocr/localocr
$ cd localocr

# configure
$ cp .env.example .env
$ nano .env    # add vision API keys

# run
$ docker-compose up -d

# open → http://localhost:8080
