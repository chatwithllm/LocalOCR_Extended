section 05 · the short version
FAQ

Still curious?

Ask in Discord — answered by the people who wrote the code. No ticket queue, no first-line bot, no escalation theater.

Is LocalOCR secure?

Everything runs on your hardware. The only external call is to the vision model (Gemini / OpenAI / Anthropic) you configured — and only for the OCR step. No analytics, no telemetry, no auth provider, no third-party CDN.

Who is LocalOCR for?

Households tracking groceries and bills, homelabbers who like clean dashboards, and small ops teams who want receipt OCR without sending images to a SaaS. If you run anything in Docker today, you can run this.

Do you charge any hidden fees?

None. The codebase is MIT. You pay your chosen vision API by usage (typically < $0.40/mo for 1,000 receipts) or skip them entirely and run Ollama LLaVA locally for $0.

How long does it take to set up?

About five minutes. git clone, copy .env.example to .env, drop in one vision API key (or none), then docker-compose up -d. Done.

Can I connect my existing receipts and bank accounts?

Yes. Drop a folder of receipt photos to backfill. Use the Telegram bot to forward new ones. Plaid integration is optional and runs locally — your bank credentials never leave your box.
