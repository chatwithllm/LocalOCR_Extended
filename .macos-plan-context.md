# Context Bundle for MACOS_APP_PLAN Agents

## Pre-gathered facts (verified)

### Tech stack
- **Backend**: Python 3 + Flask 3.1.0, gunicorn 23, SQLAlchemy 2.0.36, Alembic 1.14.1, Flask-SocketIO 5.4.1, APScheduler 3.10.4, python-telegram-bot 21.10
- **DB**: SQLite (file-based), Alembic migrations under `alembic/versions/` (33 migrations, `001_initial_schema.py` through latest)
- **Frontend**: Vanilla JS/HTML/CSS, no framework, no bundler. Single-page app: `src/frontend/index.html` (40,050 lines including embedded JS/CSS) + new `src/frontend/features.html` (676 lines, documents the 30 features) + `src/frontend/features-data.js` (1,047 lines)
- **Auth**: Flask sessions, FERNET_SECRET_KEY encrypts secrets; Google OAuth via authlib; trusted device pairing; QR-share access links
- **API style**: REST (cookie-auth)
- **Realtime**: Flask-SocketIO (limited use) + MQTT publish (paho-mqtt) for IoT push events
- **Storage**: Local filesystem for receipt images (`/data/receipts/<household>/...`)
- **OCR providers**: Gemini, OpenAI, Anthropic, Ollama (multi-model routing via `route_ai_inference.py`)
- **Integrations**: Plaid (bank/card transactions), Telegram Bot (webhook), MQTT broker, OpenRouter
- **Build tooling**: None (no webpack/vite/bundler)
- **CSS**: Inline + CSS variables (dark-theme palette `#111113`, `#1a1a1e`, `#2a2a2e`, `#3b82f6`, `#2fa36b`, `#f59e0b`)

### Frontend structure
Single-page app navigates via `nav(page, this)` (in `index.html`) which swaps content into `#main-content`. Logical screens enumerated in `src/frontend/features-data.js` — 30 features across 8 groups:

**Receipts (4)**: OCR Upload, Review & Edit, Re-run OCR, Receipt Types
**Grocery (4)**: Inventory, Shopping List, Recommendations, Kitchen View
**Restaurant (3)**: Restaurant Workspace, Repeat Orders, Dining Budget
**Expenses (3)**: Expense Tracking, Category Tagging, Expense Analytics
**Finance (4)**: Spending by Category, Fixed Bills, Plaid Integration, Cash Transactions
**Shared Dining (3)**: Split Bills, Contacts, Balances & Settle
**Telegram Bot (4)**: Shopping Walk, Inventory Walk, Dining Walk, Nudges
**Household (5)**: Auth & Members, Contributions, Demo Mode, AI Chat, Medications

Plus the `/features` page itself (separate file `features.html`).

### Constraints for the macOS app
- macOS 13 Ventura+, Apple Silicon (M1/M2/M3/M4)
- Backend stays as-is (Flask + SQLite) — macOS app is a client to the existing API
- Auth must use Keychain (NOT UserDefaults)
- Must preserve all business rules (household scoping, contribution events, attribution tagging, role gates)
- Cookie auth means the macOS HTTP client needs cookie jar + CSRF parity with web
